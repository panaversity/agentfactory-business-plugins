# Sprint Planning → Redirect

This request is about **sprint planning** — deciding which stories to pull into a sprint based on team velocity. That's a different workflow from prioritisation.

The prioritisation skill handles **strategic prioritisation**: ranking features, evaluating backlog items, framework scoring (RICE, Value vs. Effort, MoSCoW, Kano).

For sprint-level work allocation — story point math, capacity planning, sprint commitment — use **`/sprint-planning`** instead.

If you'd like to **prioritise your backlog first** (decide which items are most important) and then take the prioritised list into sprint planning, I can help with the prioritisation step.
