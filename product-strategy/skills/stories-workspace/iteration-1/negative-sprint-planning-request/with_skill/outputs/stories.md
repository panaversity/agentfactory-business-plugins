## Sprint Planning Is Outside This Skill's Scope

The `/stories` skill decomposes specs and PRDs into user stories. It does **not** handle sprint planning — that includes velocity tracking, story point budgeting, sprint commitment, and pulling stories from the backlog.

### What You Need

For sprint planning (velocity-based story selection, sprint commitment, capacity allocation), use the official **`/sprint-planning`** command instead.

### What `/stories` Can Help With

If you need to **generate, refine, or split user stories** before they enter the backlog — that's where this skill fits:

- Break a feature spec or PRD into properly sized user stories
- Add or improve acceptance criteria (Given/When/Then)
- Split oversized stories into sprint-completable slices
- Ensure stories use named personas with capability-based "wants" and user-outcome "so thats"

### Suggested Next Step

Run `/sprint-planning` with your velocity of 42 points and your current backlog to plan the sprint.
