# Knowledge Management Frameworks

## Nonaka & Takeuchi SECI Model (1995)

The SECI model describes four modes of knowledge conversion between tacit and explicit forms:

| Mode                | From     | To       | Method                                              | Example                                                       |
| ------------------- | -------- | -------- | --------------------------------------------------- | ------------------------------------------------------------- |
| **Socialisation**   | Tacit    | Tacit    | Shared experience, observation, apprenticeship      | Junior account manager shadows senior before departure        |
| **Externalisation** | Tacit    | Explicit | Articulation through dialogue, metaphor, models     | Knowledge capture interview — "tell me how this really works" |
| **Combination**     | Explicit | Explicit | Systematising, categorising, reorganising documents | Structuring interview notes into knowledge articles           |
| **Internalisation** | Explicit | Tacit    | Learning by doing, applying documented knowledge    | Successor reads knowledge articles and applies in practice    |

### How /knowledge Maps to SECI

The `/knowledge` skill operates primarily in the **Externalisation** mode — converting tacit knowledge (what the departing employee knows but has never written down) into explicit knowledge (structured knowledge articles).

The three-session interview structure is designed to facilitate externalisation:

- Session 1 (Client relationships): Externalises relational tacit knowledge
- Session 2 (Methodology): Externalises procedural tacit knowledge
- Session 3 (Institutional context): Externalises the deepest tacit knowledge — organisational wisdom

## APQC Knowledge Management Maturity Model

APQC defines five maturity levels for organisational knowledge management:

| Level | Name            | Description                                                       |
| ----- | --------------- | ----------------------------------------------------------------- |
| 1     | **Initiate**    | Ad hoc; no formal KM processes; knowledge loss happens silently   |
| 2     | **Develop**     | Pilot projects; some awareness; reactive capture on departures    |
| 3     | **Standardise** | Formal KM processes; roles defined; tools in place                |
| 4     | **Optimise**    | KM embedded in workflows; metrics tracked; continuous improvement |
| 5     | **Innovate**    | KM drives strategic advantage; predictive knowledge gap analysis  |

### Where Most Organisations Sit

Most organisations operate at Level 1-2. They react to knowledge loss after it happens (a departure) rather than systematically preventing it. The `/knowledge` skill helps move from Level 1 (ad hoc) to Level 2-3 (structured, repeatable capture).

### Progression Indicators

- **Level 1 to 2**: Organisation recognises knowledge loss as a risk; begins capture on departures
- **Level 2 to 3**: Annual knowledge audits for high-risk holders; standardised article format; assigned ownership
- **Level 3 to 4**: Knowledge capture embedded in offboarding workflow (automated triggers); article quality reviews; usage tracking

## Practical Application

When using `/knowledge`, the maturity model helps frame recommendations:

- For Level 1 organisations: Focus on immediate reactive capture for the departing employee
- For Level 2-3: Recommend proactive annual capture for all HIGH-risk holders
- For Level 4+: Recommend integration with HRIS triggers and automated scheduling
