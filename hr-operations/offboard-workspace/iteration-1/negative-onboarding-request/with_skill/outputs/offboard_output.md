# Offboard Skill — Request Declined

## Why This Request Was Not Processed

The `/offboard` skill handles **offboarding processes and knowledge transfer** for departing employees — resignations, redundancies, retirements, exit interviews, handover plans, and systems access removal.

Your request is for **onboarding a new developer**: first week schedule, equipment setup, and introductions. This falls outside the scope of the offboard skill.

From the skill description:

> NOT for: onboarding plans (use onboarding)

## What You Should Use Instead

Use the **`/onboard`** skill (or equivalent onboarding skill in your HR operations plugin) for:

- New hire first week schedules
- Equipment and systems setup checklists
- Introduction plans and buddy assignments
- Orientation programme design
- Probation period milestones

## Summary

| Requested Task | Correct Skill | Reason |
|---|---|---|
| Onboarding plan for new developer | `/onboard` (not `/offboard`) | Offboard is scoped to departing employees only |

*The `/offboard` skill correctly declined this request because onboarding is explicitly listed as an out-of-scope use case.*
